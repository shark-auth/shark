## Design Context

### Users
Server administrators and developers deploying self-hosted auth infrastructure. Technical, time-pressed, managing auth for their own products. Dashboard used in focused work sessions on desktop in dark environments. They chose SharkAuth because they want control, not hand-holding.

### Brand Personality
**Bold. Elegant. Fast. Monochrome.**

"No bullshit." Single binary, no vendor lock-in. The dashboard is a precision instrument: sharp, confident, information-dense. Every pixel earns its place.

### THE GOLD STANDARD: users.tsx
**`admin/src/components/users.tsx` is the canonical reference.** Every new surface MUST match its:
- Density (compact rows, ~32-36px row height, 7-10px padding)
- Border radii (3-5px, never higher)
- Color usage (almost zero color — hairlines + neutrals only, color reserved for *status meaning*)
- Drawer pattern (right-side fixed, 380-440px, hairline left border)
- Typography (13px base, mono for IDs/tokens only)

Other gold-standard refs (same DNA): `agents_manage.tsx`, `sessions.tsx`, `vault_manage.tsx`, `overview.tsx`.

**If a new surface doesn't visually look like it belongs in the same app as users.tsx, it's wrong. Restart.**

### CANONICAL PATTERNS (extracted from gold-standard refs, 2026-04-25)

**Split-grid list+drawer (`users.tsx`, `vault_manage.tsx`, `agents_manage.tsx`, `applications.tsx`):**
- Filter chips inline at top: `filter === v ? 'var(--surface-2)' : 'transparent'`, radius 3px, no border, weight 500 active / 400 inactive.
- Search input + `flex:1` spacer + `<count>/<total>` faint mono on right (`{filtered.length} / {providers.length}`, fontSize 11).
- Table headers `thStyle`: `padding 8px 14px, fontSize 10, fontWeight 500, color var(--fg-dim), borderBottom 1px solid var(--hairline), background var(--surface-1)`.
- Row IDs / client_ids / tokens: `<span className="mono faint" style={{fontSize:10.5}}>{truncateMiddle(id, 24)}</span>`.
- Drawer pattern: `position:fixed; top:0; right:0; bottom:0; width:380-440px; border-left:1px solid var(--hairline)`. See `users.tsx` ~line 477.

**Page-level keyboard wiring:**
- `usePageActions({ onNew: () => setCreateOpen(true), onRefresh: refresh })` wires `n` → new, `r` → refresh on every list page. Use it. Don't reinvent.
- `?` opens shortcuts overlay, `/` focuses search, `g<key>` jumps nav. Already global — don't add page-specific equivalents.

**URL-synced state:**
- `useTabParam(default)` for tabs, `useURLParams.tsx` for filters. State must survive copy/paste of URL. NEVER store list filters in `useState` only.
- `?new=1` query param opens create drawer on mount (vault, applications, agents). Strip the param after handling. This is how external links / docs deep-link into "create" flows.

**Overview page rhythm (`overview.tsx`):**
- Cold-start gate: when `users.total === 0 && !proxyConfigured`, show single magical-moment hero tile with `linear-gradient(135deg, var(--surface-2), var(--surface-1))` + `1px solid var(--hairline-strong)`. Exception to monochrome rule — gradient only on this one tile, only on cold-start. Persists dismiss to `localStorage('shark_hide_hero')`.
- After cold-start: KPI grid → attention panel (degraded health, expiring keys, errors needing attention) → SSE live activity stream → auth method breakdown.
- Live activity uses `EventSource('/api/v1/admin/logs/stream')` with exponential backoff (`Math.min(1000 * 2^retry, 30000)`). Status badge: `connecting | live | reconnecting`.
- "Skip, show me the dashboard" is a `btn ghost sm` — never primary on a dismiss.

**Vault rhythm (`vault_manage.tsx`):**
- Provider rows: brand icon (22px) → display_name bold → mono-faint name → mono-faint truncated client_id → scope count → status dot. No row > 36px.
- "Connect provider" creates from a slide-over, NOT a centered modal. Modal == lazy.
- Secret values: never display. Show `••••` or `<reveal>` with copy-on-click button (`CopyField` from `shared.tsx`).
- Audit log lives in the drawer, not a separate page.

### MONOCHROME RULE (HARD)
- B&W foundation. Surfaces step `#000` → `#0d0d0d` → `#141414` → `#1c1c1c`.
- **NO colored brand accents.** No purple, blue, teal, gradient anything.
- Color appears ONLY when it carries status meaning:
  - `var(--success)` — running / healthy / done
  - `var(--warn)` — degraded / pending / unsaved
  - `var(--danger)` — error / destructive / stopped
  - `var(--agent)` — only for agent-vs-human distinction in identity surfaces
- Use color in tiny doses: 6-8px status dots, 1px hairline borders, single text color on a label. Never as a background fill on a button, never as a tab underline color, never as a card border.
- If you find yourself reaching for a colored card, stop and use a hairline border instead.

### SQUARE RULE (HARD)
- Border radius capped at **3-5px**. Never 6+, never pill, never circle (status dots and avatars only).
- Sharp 90° corners on rows, tables, sections, panels.
- No "rounded card" aesthetic. No 12px+ radius anywhere.

### EDITABLE CONFIG RULE (HARD)
- A config page where fields are read-only is broken. **Every value an admin sees on a config screen must be editable from that screen** unless it's a calculated/derived state (uptime, version, build SHA, status).
- Read-only "info panels" with config values are a smell. Replace with inputs.
- Pre-existing values populate the inputs; admin edits in place; explicit Save commits with API call.
- Show dirty-state indicator + Discard / Save bar.

### NO SESSION vs JWT (HARD)
- SharkAuth uses BOTH cookie sessions (`shark_session`, AES+HMAC) AND JWTs (signing keys, OAuth tokens) — **both always enabled, simultaneously, with no toggle.**
- There is NO "session mode" or "JWT mode" choice anywhere. NEVER expose them as competing options, modes, tabs, or radio choices.
- A session is the auth context (cookie). A JWT is a signed token derived from / linked to that session for OAuth/M2M/SDK flows.
- Config surfaces these as ONE concept: "Session & Token" with cookie lifetime + JWT signing key + JWT lifetime as ordinary fields next to each other.

### NO YAML EXPOSURE (HARD)
- `sharkauth.yaml` is being **deprecated**. Do NOT add YAML import/export/preview UIs.
- Do NOT show "view as YAML" toggles, code-block YAML previews, or YAML drag-drop import drawers.
- Configure via UI fields → backend writes config via admin API. YAML is implementation detail.

### HIGH-LEVEL CONFIG ONLY
- Surface what admins should *care* about. Not every koanf field.
- Examples of HIGH-LEVEL (good): "Session lifetime: 7 days", "Allow password login: ✓", "Magic-link redirect after login: /dashboard"
- Examples of LOW-LEVEL (avoid): "JWT iss claim", "JWT clock skew (seconds)", "Cookie SameSite policy", "AES block-key derivation algorithm"
- If a config has only one sensible value, hide it. If it requires reading docs to set safely, hide it. Surface the 80%, hide the 20%.

### Component Shape Philosophy
- **Reference pages (gold standard):** `users.tsx`, `agents_manage.tsx`, `sessions.tsx`. Copy their exact rhythm.
- **Anti-reference pages (rebuilt 2026-04-24, must not regress):** `proxy_wizard.tsx`, `proxy_config.tsx` (rebuilt), `get_started.tsx` (rebuilt), `Walkthrough.tsx`, `identity_hub.tsx` (rebuilt), `settings.tsx` (rebuilt). The first rebuild attempt FAILED — too colorful, too many read-onlys, exposed JWT-vs-session, exposed YAML. THIS REBUILD MUST FIX ALL OF THAT.
- **Drawer pattern:** `position: fixed; top: 0; right: 0; bottom: 0; width: 380-440px; border-left: 1px solid var(--hairline)`. Square edges. See `users.tsx` ~line 477.
- **Sizes:** Cards radius 4-5px. Inputs radius 4px. Buttons radius 4-5px. Badges 3px. Avatars/status dots circular only.
- **Padding:** 8-16px inside cards. 7-10px row padding. NEVER `padding: 24px` for chrome.

### Typography Direction
- **Display:** Hanken Grotesk
- **Body:** Manrope (13px base on dark)
- **Mono:** Azeret Mono (IDs, tokens, keys only — not for "technical vibe")
- Fixed rem scale (this is product UI, not marketing). Min 1.25 ratio between steps.

### Design Principles
1. **Match users.tsx.** If it doesn't look like users.tsx, it's wrong.
2. **Monochrome.** Color is for status, never decoration.
3. **Square.** Sharp corners, hairline borders, low radius.
4. **Editable.** Config means inputs, not displays.
5. **Cookie + JWT both always on.** Never expose as choice.
6. **No YAML in UI.** Deprecated path.
7. **High-level only.** Hide knobs that need docs to set.
8. **Density over decoration.** Show data, not chrome.
9. **Drawer over modal.** Right-side fixed panel for inspect/edit.
10. **Speed as UX.** Optimistic mutations, skeletons, keyboard shortcuts.
