# Migration 00024 — proxy_rules.m2m

## SQL

```sql
-- +goose Up
-- PROXYV1_5 §4.17: add m2m boolean so rules can be locked to agent
-- callers (service-to-service). Default 0 keeps legacy rules human-inclusive.
ALTER TABLE proxy_rules ADD COLUMN m2m INTEGER NOT NULL DEFAULT 0;

-- +goose Down
ALTER TABLE proxy_rules DROP COLUMN m2m;
```

## Purpose

Adds the boolean column that backs `RuleSpec.M2M` — a rule-level flag that restricts the rule to agent-typed callers. See `contracts/m2m_rule_flag.md` for full semantics.

The column is encoded as `INTEGER NOT NULL DEFAULT 0` rather than a native boolean because SQLite doesn't carry one; the Go storage layer round-trips through `boolToInt` / `intToBool` so the application code sees a clean `bool`.

## Column details

- Name: `m2m`
- Type: `INTEGER NOT NULL DEFAULT 0`
- Encoding: `0 = false` (rule accepts human or agent callers), `1 = true` (rule requires agent).
- Default: `0`. Pre-existing rules remain human-inclusive on upgrade — the flag is additive.

## Consumers

### Writers

- `POST /api/v1/admin/proxy/rules/db` — accepts `m2m` in the create body.
- `PATCH /api/v1/admin/proxy/rules/db/{id}` — accepts `m2m` as an optional pointer field.
- `POST /api/v1/admin/proxy/rules/import` — YAML import recognises `m2m: true`.

### Readers

- `Store.ListProxyRules` + `Store.GetProxyRuleByID` return the column via the shared select list in `proxy_rules_sqlite.go`. `finalizeProxyRule` is the single choke point for the int-to-bool conversion.
- `refreshProxyEngineFromDB` copies `r.M2M` into `proxy.RuleSpec.M2M`. `compileRule` mirrors it onto the compiled `Rule`. `Engine.Evaluate` enforces the gate before evaluating `Require`.

## Wire shape

Rule response JSON always includes `"m2m": <bool>` (not omitempty) so clients don't have to distinguish absent from false. Create + update accept the field optionally; absence preserves the current value.

## Audit

Evaluating an M2M-gated rule with a human-typed caller produces the distinct reason string `"rule requires agent (m2m)"` in `Decision.Reason`, and the proxy audit log records `actor_type=human` alongside the deny. The combination is a clean filter for "phished human session attempted an M2M-only surface".
