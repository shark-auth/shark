# Shark Proxy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Zero-code auth for any backend. Shark sits in front of upstream apps, validates auth (JWT/cookies), injects identity headers. Circuit breaker ensures auth server failure never kills the proxy.

**Architecture:** Two modes — embedded (`shark serve --proxy-upstream`) and standalone (`shark proxy --upstream`). Same binary, same code. Auth routes always bypass proxy. Route-level rules engine for fine-grained access control.

**Tech Stack:** Go + chi + httputil.ReverseProxy + LRU cache (circuit breaker)

**Depends on:** Phase 5 (OAuth 2.1) — JWT verification for agent tokens

---

## Core Architecture

```
shark serve --proxy-upstream http://localhost:3000

├── /api/v1/*        → Shark auth handlers (NEVER proxied, always alive)
├── /oauth/*         → OAuth handlers (NEVER proxied)
├── /admin/*         → Dashboard (NEVER proxied)
├── /.well-known/*   → Discovery (NEVER proxied)
└── /*               → Proxy middleware → auth check → inject headers → upstream
                        ↓ panic/crash/timeout
                     recover() → log → 503 (Shark keeps serving)
```

## Circuit Breaker Design

| Layer | Mechanism | Covers |
|-------|-----------|--------|
| L1: JWT local verify | Cached JWKS, stateless ES256 sig check | Agent/OAuth flows |
| L2: Session decision cache | LRU: cookie_hash → {user, roles, ttl} | Human sessions |
| L3: Health monitor | Ping /healthz every 10s, 3 fails → circuit open | Auto-switch |
| L4: Degraded headers | X-Shark-Auth-Mode, X-Shark-Cache-Age | Upstream awareness |

**Key guarantee:** Agents with JWT tokens NEVER experience auth downtime (verification is local math).

    rules:

### Task 3: Circuit Breaker
- [x] Implement `internal/proxy/circuit.go`:
  - LRU cache for session auth decisions (configurable size, default 10K entries)
  - TTL per cache entry (configurable, default 5min)
  - Health monitor goroutine (ping /healthz every 10s)
  - State machine: closed → open (3 failures) → half-open (1 success) → closed
- [x] When circuit open:
  - JWT tokens: still verified locally (JWKS cached) — no degradation
  - Cookie with cache hit: serve from cache, add `X-Shark-Auth-Mode: cached`
  - Cookie with cache miss: configurable (reject or allow-read-only)
  - Add `X-Shark-Cache-Age: N` header (seconds since last live verification)
- [x] Negative cache: bad tokens cached for 30s to prevent hammering
- [x] Write tests (state transitions, cache hit/miss, health monitor)
- [x] Commit
