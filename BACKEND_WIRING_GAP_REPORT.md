# Backend Wiring Gap Report

**Date:** 2026-04-22
**Status:** Comprehensive Audit Complete
**Objective:** Document all backend capabilities NOT wired into the frontend dashboard.

---

## 1. User & Access Management (RBAC, Users, Orgs)

### Gaps in User Profile & Security
*   **Linked OAuth Accounts:** Backend supports listing and deleting user OAuth connections (`GET/DELETE /users/{id}/oauth-accounts`), but this is entirely missing from the admin UI.
*   **Passkey Management:** Backend supports listing passkeys (`GET /admin/users/{id}/passkeys`), but the UI does not display them.
*   **MFA Metadata:** `MFAVerifiedAt` timestamp (tracking the first successful verification) is available in the backend but not displayed in the dashboard.
*   **Avatar Management:** `avatar_url` is supported in the User struct but the UI lacks an update mechanism.

### RBAC & Organizations
*   **Reverse Permission Lookup:** Backend supports \"Who has this permission?\" via `GET /permissions/{id}/roles` and `/users`, but the UI lacks this drill-down.
*   **Permission Management UI:** No dedicated UI to manage the global permission list; permissions are only managed via role attachment.
*   **Org Role Granularity:** UI for Organization Roles lacks permission mapping; backend supports `AttachOrgPermission` but UI only manages role name/description.
*   **Member Management:** Missing wiring for `handleUpdateOrganizationMemberRole` and `handleGrantOrgRole` in the members tab.

---

## 2. OAuth, SSO & Identity Flows

### SSO Gaps
*   **SAML Configuration:** SAML SP Metadata and ACS URLs are generated by the backend but not displayed in the dashboard, hindering manual IdP setup.
*   **Attribute Mapping:** Advanced SAML Attribute Mapping and OIDC scope configurations in the backend lack UI inputs.
*   **Auto-Routing Logic:** Frontend implements domain-to-connection routing client-side, bypassing the backend's `SSOAutoRoute` resolution logic.

### Advanced OAuth
*   **Dynamic Client Registration (DCR):** No UI to manage DCR-created clients or rotate Registration Access Tokens (RATs).
*   **Token Exchange (RFC 8693):** Fully implemented in backend for agent-to-agent delegation, but entirely missing from the dashboard.
*   **DPoP Enforcement:** No UI configuration to enforce sender-constraining (DPoP) at the Agent or Application level.
*   **Device Flow Overrides:** Admin UI for device flow approval lacks the backend's ability to bind a specific `user_id` during override.
*   **JTI Revocation:** No interface for managing the global JTI revocation (blacklist) table.

---

## 3. Infrastructure (Proxy, Webhooks, Vault)

### Proxy & Webhooks
*   **Proxy Status Real-time:** Backend supports SSE for live status, but the dashboard uses legacy polling.
*   **Proxy Simulator:** Missing `user_email` and `agent_id` fields in the simulation UI despite backend support.
*   **Webhook Analytics:** Frontend expects `success_rate_7d` and `last_delivery_at` in the list view, but the current backend response does not provide these computed fields.
*   **Webhook Pagination:** Backend supports pagination; frontend lacks the UI controls to navigate large delivery histories.

### Vault
*   **Scope Overrides:** Backend supports overriding OAuth scopes during the connection flow, but the UI lacks an input for this.
*   **Metadata Visibility:** Connection audit metadata is not fully exposed in the provider detail view.

---

## 4. System, Audit & Configuration

### Audit Logs
*   **Filter Mismatch:** Backend supports 12+ filters (including `session_id`, `resource_id`, `ip`); UI only exposes 4 basic filters.
*   **Retention Management:** `StartCleanup` worker exists in backend, but there is no UI to configure the retention period.
*   **Audit Sinks:** \"Send to sink\" and \"Replay\" buttons are placeholders not wired to the backend event streaming.

### Telemetry & Stats
*   **Trend Data:** Overview trends for sessions, MFA, and agents are mocked because the backend `/admin/stats/trends` currently only provides signups.
*   **SSO Breakdown:** `sso_connections.user_counts` (identities per provider) is returned by the API but not displayed.

### Configuration & Health
*   **Email Template Editor:** Backend supports full CRUD for email templates, but the UI only provides a static preview.
*   **Password Policy:** Complexity requirements are returned as read-only in the UI and hardcoded in the backend; no configuration interface exists.
*   **Dev Inbox:** The backend \"Dev Mode\" email inbox is not accessible via any dashboard view.
*   **SMTP Tracking:** `sent_today` and `daily_limit` are present in the API schema but not yet implemented/tracked in the backend logic.

---

## Summary of Impact
The dashboard currently acts primarily as a **monitoring tool** for many features where the backend is already a **management engine**. High-priority wiring is needed for **SAML Metadata visibility**, **Proxy Rule CRUD**, and **User Passkey/OAuth Management** to achieve launch readiness.
